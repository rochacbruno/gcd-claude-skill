# Service Usage overview 



Service Usage is an infrastructure service of Google Cloud Dedicated in Germany that lets
you list and manage APIs and services in your Google Cloud Dedicated projects.

Before you can use most Google APIs you must first enable them in a
Google Cloud Dedicated project. You can use Service Usage to perform the
following tasks:

- [List enabled services in an organization or project](/service-usage/docs/list-services)

- [Enable or disable services for a Google Cloud Dedicated project](/service-usage/docs/enable-disable)

- [Enable or disable services at different levels of the resource hierarchy](/service-usage/docs/hierarchical-service-activation/overview) (Preview)

Some APIs and services are enabled by default when you create a
Google Cloud Dedicated project, while others are automatically enabled when you use them
in the Google Cloud Dedicated console. For more information, see
[Enabled services](/service-usage/docs/enabled-service).

You can use Service Usage through the
[Google Cloud Dedicated console](https://console.cloud.berlin-build0.goog/), from the
[Google Cloud CLI](/sdk/gcloud/reference/services), or programmatically
using the [Service Usage API](/service-usage/docs/reference/rest) from
one of the [Google API Client Libraries](/service-usage/docs/libraries).
